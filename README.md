# 🧸 Pediatric Dental World

> **Gentle Care, Happy Smiles.**
> A premium, highly polished, fully responsive static frontend website designed for a modern pediatric dental clinic, integrated seamlessly with a serverless **Supabase** backend.

---

## 🌐 Architecture Overview

This application is built with a decoupled, high-performance, and serverless architecture:

1. **Frontend (Static Hosting)**
   - **Tech Stack:** Semantic HTML5, modern CSS3 (with responsive layouts), and vanilla JavaScript.
   - **Deployment:** Directly hostable as a static site on **GitHub Pages**, Netlify, Vercel, or simply opened locally by double-clicking `index.html`.
   - **Build Step:** None. Zero npm compile steps, zero render-blocking bundle code, zero local server prerequisites.

2. **Backend (Serverless Database)**
   - **Tech Stack:** Powered entirely by **Supabase** (Postgres DB) via the official JavaScript SDK loaded dynamically from a high-speed CDN.
   - **Functions:** Stores appointment requests, contact form messages, and newsletter health-tip subscriptions instantly into real-time Postgres tables.
   - **Authentication/Security:** Secures public client submissions via standard Row Level Security (RLS) insert-only policies.

## ⚡ Step-by-Step Supabase Backend Setup

To connect your static frontend to your live database, follow these simple steps:

### Step 1: Create Your Supabase Project
1. Log in to [Supabase](https://supabase.com) (free tier is fully supported).
2. Click **New Project** and select your region/pricing plan.

### Step 2: Initialize Database Tables (Copy & Paste SQL)
Go to the **SQL Editor** tab in your Supabase dashboard, click **New Query**, paste the following script, and click **Run**:

```sql
-- 1. Create Appointments Table
CREATE TABLE IF NOT EXISTS public.appointments (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    parent_name TEXT NOT NULL,
    child_name TEXT NOT NULL,
    child_age INT NOT NULL,
    phone TEXT NOT NULL,
    email TEXT NOT NULL,
    preferred_date DATE NOT NULL,
    preferred_time TEXT NOT NULL,
    service_needed TEXT NOT NULL,
    doctor_preference TEXT,
    notes TEXT
);

-- 2. Create Contact Submissions Table
CREATE TABLE IF NOT EXISTS public.contact_submissions (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    name TEXT NOT NULL,
    email TEXT NOT NULL,
    phone TEXT NOT NULL,
    subject TEXT NOT NULL,
    message TEXT NOT NULL
);

-- 3. Create Newsletter Subscribers Table
CREATE TABLE IF NOT EXISTS public.newsletter_subscribers (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    email TEXT UNIQUE NOT NULL
);

-- 4. Enable Row Level Security (RLS) on all tables
ALTER TABLE public.appointments ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.contact_submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.newsletter_subscribers ENABLE ROW LEVEL SECURITY;

-- 5. Create INSERT-ONLY public policies so frontend forms can insert data securely
CREATE POLICY "Allow anonymous appointment inserts" ON public.appointments
    FOR INSERT WITH CHECK (true);

CREATE POLICY "Allow anonymous contact inserts" ON public.contact_submissions
    FOR INSERT WITH CHECK (true);

CREATE POLICY "Allow anonymous newsletter inserts" ON public.newsletter_subscribers
    FOR INSERT WITH CHECK (true);
```

### Step 3: Insert API Key Credentials
1. In Supabase, go to **Project Settings** (gear icon) -> **API**.
2. Locate your **Project URL** (e.g. `https://xyz.supabase.co`) and **anon public** key.
3. Open `supabase-config.js` in your editor and paste them into their respective placeholders:

```javascript
const SUPABASE_URL = "https://YOUR-SUBDOMAIN.supabase.co"; 
const SUPABASE_ANON_KEY = "YOUR-PUBLIC-ANON-API-KEY";
```

## 🎯 High-Performance UX Features

- **Safe Demo Fallback Mode:** If the `supabase-config.js` credentials are left as placeholders, the application automatically runs in **Demo Mode**. All form submissions are simulated locally and safely cached in `localStorage` to allow flawless prototyping and live demonstrations.
- **Urgency Booking Action:** Once a parent books, they receive an immediate visual confirmation along with an active, high-visibility **WhatsApp CTA** to instantly redirect them to the clinic's support line for scheduling confirmation or dental emergencies.
- **Responsive Fluidity:** Leverages clean fluid containers and CSS grids to adapt seamlessly across mobile phones, tablets, foldables, and large desktop screens.
- **Local SEO Optimized:** Out-of-the-box structured schema.org JSON-LD microdata inserted into `index.html` to guarantee the clinic ranks perfectly on search engines for pediatric keywords.
